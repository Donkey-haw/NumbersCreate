# TSD 수정 기반 넘버스 파일 생성 실행 계획

이 문서는 `Plan.md`에서 정의한 파이프라인을 실제로 구현하고 검증하기 위한 구체적인 단계별 프로세스와 마일스톤을 제시합니다.

## 페이즈 1: 프로젝트 초기화 및 환경 구성
**목표**: 스크립트 실행 환경 구축 및 필요 라이브러리 설치
- [ ] Python 가상환경(venv) 생성 및 활성화
- [ ] `snappy` 압축 처리를 위한 패키지 설치 (`cramjam`, `python-snappy` 등)
- [ ] Protobuf 컴파일러(`protoc`) 설치 및 Python 패키지(`protobuf`) 설치
- [ ] IWA 스키마 확보: `m-tymchyk/apple-iwork-protos` 또는 `numbers-parser` 내의 `.proto` 파일 컴파일 및 Python 모듈화

## 페이즈 2: IWA 파일 파싱 코어 개발
**목표**: `.numbers` 패키지에서 IWA 파일을 추출하고 내용을 읽어오는 기능 구현
- [ ] ZIP 압축 해제 모듈 작성 (`DocumentExtractor`)
- [ ] `.iwa` 파일을 입력으로 받아 Snappy 청크를 디코딩하는 함수 작성
- [ ] 디코딩된 청크를 Protobuf `TSP.ArchiveInfo` 메시지로 역직렬화하는 모듈 작성
- [ ] **테스트**: "빈 넘버스 파일.numbers"의 `Index/Document.iwa` 파일을 성공적으로 파싱하여 모델 객체로 가져오는지 확인

## 페이즈 3: TSD 데이터 매핑 및 수정 로직 구현
**목표**: 객체의 좌표를 담고 있는 Geometry 데이터를 찾고 수정하기
- [ ] 참조 그래프(Reference Graph) 검색 유틸리티 작성
- [ ] 타겟 객체(예: 테이블, 텍스트 박스, 이미지 이미지)의 `TSD.GeometryArchive` 찾기
- [ ] `origin(x, y)` 및 `size(w, h)` 값을 동적으로 변경하는 코드 구현
- [ ] 변경된 `GeometryArchive`를 원본 객체 구조에 반영 및 Protobuf로 다시 직렬화

## 페이즈 4: 패키징 및 최종 파일 생성
**목표**: 수정된 데이터를 다시 `.numbers` 파일로 조립
- [ ] 수정된 바이너리에 IWA 청크 헤더(길이 정보 포함)와 Snappy 압축을 다시 적용하는 인코더 작성
- [ ] 덮어씌워진 `.iwa` 파일들을 포함하여 임시 폴더의 모든 내용을 ZIP으로 압축
- [ ] 결과를 `modified.numbers`로 저장
- [ ] **테스트**: 생성된 파일이 Mac의 Numbers 앱에서 렌더링 오류 없이 제대로 열리는지, 좌표나 크기가 정상적으로 변경되었는지 육안 검증

## 페이즈 5: 추상화 및 자동화 스크립트 테스트
**목표**: 사용 가능한 형태로 CLI 또는 Python 유틸리티 스크립트화 확인
- [ ] 1~4단계를 통합하는 메인 스크립트 `generate_numbers.py` (또는 유사 스크립트) 작성
- [ ] 커맨드라인에서 입력 파일과 수정할 좌표 파라미터를 입력할 수 있게 구성
- [ ] (옵션) 기존 `.numbers` 내의 템플릿의 데이터를 채우고 일괄 생성하는 배치 기능 지원
